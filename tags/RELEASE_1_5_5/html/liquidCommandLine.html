Liquid: Command Line Reference

Liquid is implemented as a Maya command so it is easily scriptable and all of the render globals are accessable through the command line interface.

General
-GL use the liquid render globals, this should ALWAYS come first!
-sel render the currently selected objects.  This excludes lights, all current visible lights will still render.
-p preview mode, will write out a preview rib file and render it.
-d debug mode - will print debug information to maya's Output Window
-err error checking on

Frame
-n render a range -n firstframe lastframe step
-m set the number of motion samples up to 5
-dofOn include camera's Depth of Field information.
-x x resolution
-y y resolution
-aspect pixel aspect ratio
-mb transformation blur
-db deformation blur

File
-mf input maya file - full path name.
-tiff forces liquid to output .tiff files.
-pad set outpadding amount (if any)
-ribName rib output name
-od the project directory
-picdir image directory
-texdir texture/shadow directory
-ribdir rib directory
-preFrameMel "scriptName" scriptName is the name of a mel script that will be run before liquid generates any output at the current frame sample.
-postFrameMel "scriptName" scriptName is the name of a mel script that will be run after liquid generates any output at the current frame sample.

Image
-s pixel samples
-sr shading rate
-bs x y bucketsize
-gs gridsize
-texmem texture memory
-es eyesplits

Network
-alfred use alfred to render
-noalfred if for some reason liquid is trying to call alfred, this forces it not to
-remote remote render if alfred is being used.
-netRman if alfred is being used then netrender the frames

Options
-allCurves force liquid to output all nurbs curves as RiCurves.
-doBinary output binary ribs.
-doCompression output compressed ribs.
-shadows render and use shadows
-noshadows force shadow rendering off
-fullShadowRib force full shadow rib generation
-cleanRib clean up rib files upon completion
-progress liquid will output the progress of the generation.  ( command line liquid will output the progress in a format that controls alfred progress meter. )
-shaderDB shader debug mode, if an object is missing an assigned renderman shader it will be made bright red.

BMRT
-r renderer using - 0 = PRMan, 1 = BMRT
