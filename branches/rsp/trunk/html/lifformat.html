lif: Format Reference

Introduction
    The lif ( liquid interface file ) files are used to control the appearance of shader parameter editing to make it more intutive for artists to control the parameters, once these files have been created they have to be located in the same directory as the shader to work.  These files are read every time the parameter dialog box is opened so if changes are made to the file the interface will be updated the next time the parameter dialog is opened.

The best way to introduce the structure of the files is to give an example so here is a simple lif file for a surface shader with three parameters defined.

surface "myShader";
desc "a very simple shader";

# parameter Ks

param "Ks";
desc "this control the amount of specular light reflected by the surface";
name "Specular Value";

param "Kd";
name "Diffuse Value";
desc "this control the amount of diffuse light reflected by the surface";
type "slider";
range [ 10.0 100.0 ];

param "Ka";
name "Ambient Value";
desc "this control the amount of ambient light reflected by the surface";
type "selector";
range [ "One" 1.0 "Radiance" 10.0 "Three" 3.0 ];

Comments
    Comments can be added to the lif files by adding a hash ( # ) symbol at the beginning of the line, everything from that point to the end of line is ignored.

End Of Command
    All commands should be terminated with a semi-colon ( ; ) similar to C and SL.

Key Words
surface <name>, lightsource  <name>, displacement <name>, volume <name> :

    The first line of the file should contain the type of shader and the name of the shader.

desc <description>

    When used before a parameter is defined this will be the shader's decsription, after a parameter has been defined this will be that parameters description.

param <parameter name>

    This defines the beginning of a parameter block and all the commands after this one will affect the parameter defined in the arguments.  The parameter name must be the exact name ( case sensitive ) as the parameter in the shader.

name <name>

    The name command allows you to define a more descriptive name used in the parameter window for the shader parameter.  Rather than having an artist work parameter names like Ka, TexCol they can be defined as Amient Value and Texture Color.

type <type>

    By default the parameter dialog box handles floats with sliders and strings as basic string inputs, this command allows the user to define more intuitive interfaces for the parameters.  Currently the supported types are selector and slider.  By defining a slider the user can then use the range command to define the range that the slider covers, a selector creates a drop down menu where various options can be entered and the float value will be set accordingly.

range [<min> <max>]
range [ <name> <value> <name2> <value2> ... ]

    The range command allows the user to define either the min and max values for a slider or a name/value pair for a selector.  The slider range expects two values where the selector can be any number of values.
